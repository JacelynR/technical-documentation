Introduction

Flexbox is a single dimensional layout method for arranging items in rows or columns. Items flex to fill additional spaces and shrink to fit into smaller spaces

In the past, the only reliable cross browser-compatible tools for creating CSS layouts were things like floats and positioning. These are okay, and they work, but in some ways they are limit what you can do.

The following simple layout requirements are either, difficult or impossible, to achieve with past tools, in any kind of convenient, flexible way:

  • Vertically centering a block of content inside its parent.
  • Making all the children of a container take up an equal amount of the available width/height, regardless of how much width/height is available.
  • Making all columns in a multiple-column layout adopt the same height even if they contain a different amount of content.

Flexbox makes a lot of layout tasks much easier

Flex Model

When elements are laid out as flex items, they are laid out along two axes:



  • The main axis is the axis running in the direction the items to be flexed are being laid out in (e.g. as rows across the page, or columns down the page.) The start and end of this axis are called the main start and main end.
  • The cross axis is the axis running perpendicular to the direction the flex items are being laid out in. The start and end of this axis are called the cross start and cross end. The parent element that has display: flex set on it is called the flex container.
  • The items being set out as flexible boxes inside the flex container are called flex items.

Flex Columns or Rows

Flexbox provides a property called flex-direction that specifies what direction the main axis runs in (the direction the flexbox children are laid out in) — by default this is set to row, which causes them to be laid out in a row in the direction your browser's default language works in (left to right normally).


flex-direction: column;


You'll see that this puts the items back in a column layout, much like they were before we added any CSS

Note: You can also lay out flex items in a reverse direction using the row-reverse and column-reverse values

Ordering Flex Items

Flexbox is also great for changing the layout order of flex items, without affecting the source order. This is another task that is impossible to do with traditional layout methods. 

             
               button:first-child {
                 order: 1;
               }

When you do this, this is how this works:

  • By default, all flex items have an order value of 0.
  • Flex items with higher order values set on them will appear later in the display order than items with lower order values.
  • Flex items with the same order value will appear in their source order. So if you have four items with order values of 2, 1, 1, and 0 set on them respectively, their display order would be 4th, 2nd, 3rd, then 1st.
  • The 3rd item appears after the 2nd because it has the same order value and is after it in the source order.

Cross Browser Compatibility

Flexbox support is available in most new browsers — Firefox, Chrome, Microsoft Edge, and IE 11, newer versions of Android/iOS, etc. But, you should be aware that there are still older browsers in use that don't support Flexbox (or do, as an out-of-date version of it.)

While you are just learning, this doesn't matter too much. However, if you are considering using flexbox in a real website; do testing and make sure that your user experience is still accepted in as many browsers as possible.

Flexbox is slighty difficult than some CSS features. For example, if a browser is missing a CSS drop shadow, then the site will likely still be usable. Not supporting flexbox features however will probably break a layout completely, making it unusable.
